Introduction

This is a quick introduction to Kitten. It glosses over some nuances of using the language in order to provide a fast, high-level overview. For a more complete treatment of how to use Kitten, see the tutorial.

Comments

Comments are C-style. Multi-line comments are allowed to nest. 

// Single-line comment.

/*
  Multi-line
  comment.
*/

/*
  Nested

  /* multi-line */

  comment.
*/

Literals

Integers consist of decimal digits; floating-point numbers must also include a decimal point . with a fractional part. 

6
6.28

String literals use double-quotes, and are syntactic sugar for vectors of characters. Common string escapes are allowed using backslash \. 

""
"Meow!\n"

Containers

Vectors are surrounded by square brackets [] with comma-separated elements of the same type. A trailing comma is allowed. 

[]

[1, 2, 3]

[
  "four",
  "five",
  "six",
]

Tuples are similar to vectors, using parentheses () instead. Tuples may contain elements of different types. It is possible to have a zero-element tuple, or a one-element tuple using a trailing comma. 

(1, "two", 3.0)
()
("silly",)

Expressions

Expressions and function calls use postfix syntax and are evaluated from left to right using a stack-based evaluation model. 

2 3 +  // push 2, push 3, add

"Meow!\n" print

Parentheses are allowed for readability, but they have no effect on the meaning of the code because there is no operator precedence. 

(2 3 +) (4 5 -) *

Conditionals and Blocks

Conditional execution uses the if-then-else syntax. Blocks can be surrounded by curly braces {} like in C-style languages, or prefixed with a colon : and delimited by layout. 

if foodDish isEmpty then {
  ":(" say
  begForFood
} else {
  purr
}

if foodDish isEmpty then:
  ":(" say
  begForFood
else:
  purr

if foodDish isEmpty then: ":(" say begForFood else: purr

Function Definitions

Functions are declared with the def keyword. They have a name, a type signature, and a body. Function types are written with an ASCII rightwards arrow ->. 

def square (Int -> Int):
  dup *

Local variables can be declared with the ->name syntax. This takes a value from the stack and stores it in a local variable for the duration of a block. Local variables are always immutable. 

def square (Int -> Int):
  ->x
  x x *

Functions can take multiple inputs; in the type signature, their types are separated by spaces. There is no overloading; the * operator is for Int multiplication, and *. is for Float. 

def multiply3 (Float Float Float -> Float):
  ->z
  ->y
  ->x
  x y z *. *.

To reduce stack ordering errors, multiple local variables can be taken from the stack at once in the order they were pushed. 

1 2 3 ->{ x y z }
x sayInt  // 1
y sayInt  // 2
z sayInt  // 3

def multiply3 (Float Float Float -> Float):
  ->{ x y z }
  x y z *. *.

However, it is also common to chain functions without using local variables. You can pass arguments and return values implicitly on the stack. 

def multiply3 (Float Float Float -> Float):
  *. *.

Functions may also have multiple return values. 

def incrementBoth (Int Int -> Int Int):
  ->{ x y }
  (x 1 +) (y 1 +)

Polymorphism

Type signatures often include type variables, which enable polymorphism. For example, here is a simple recursive function to reverse a vector: 

def reverse ([a] -> [a]):
  ->v
  if v isEmpty then:
    []
  else:
    v init reverse
    v last prepend

The type variable a allows this function to work on vectors of any element type. The type of map is [a] (a -> b) -> [b], that is, it takes a vector of a values, and a function from a to b, and returns a vector of b values. 

[3, 6, 9, 12] {3 /} map  // [1, 2, 3, 4]

Side Effects

When you write a function that does I/O, you must write the side effect in the type signature as +IO. The say function from the standard library has the type [Char] -> +IO—it takes a string of characters and returns nothing, plus an I/O side-effect. 

def say ([Char] -> +IO):
  print
  "\n" print

def greetAndGetName ([Char] -> [Char] +IO):
  "What is your name? " prompt ->name
  ["Pleased to meet you, ", name, "!"] concat say
  name

Effect Polymorphism

You can also write functions which are polymorphic by effect. The map function in the standard library has the type [a] (a -> b +e) -> [b] +e, which means that if its argument does some side effect e, then map also has that side effect. 

["fee", "fie", "foe", "fum"]
{
  ->word
  word say
  word "ble" cat
}
map

// Outputs:
// fee
// fie
// foe
// fum
//
// Returns:
// ["feeble", "fieble", "foeble", "fumble"]

The operator to apply a function to the stack is called @ (“apply”). Here is a function which takes two functions and returns their composition: 

def compose (
  (a -> b +e1)
  (b -> c +e2)
  ->
  (a -> c +e1+e2)
):
  ->{ f g }
  { f@ g@ }

The first function has effect e1 and the second has effect e2, so their composition has effect e1+e2. Currently, the only effects are +IO and +(), meaning “pure”—the composition of two pure functions is pure, and the composition of an impure function with any function is impure.
